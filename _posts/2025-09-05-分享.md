## podman 命令

```sh
以下是 `podman` 的常用命令分类介绍：

## 容器管理

- `podman run [OPTIONS] IMAGE [COMMAND] [ARG...]`：创建并启动一个容器
- `podman start CONTAINER`：启动一个或多个已停止的容器
- `podman stop CONTAINER`：停止正在运行的容器
- `podman restart CONTAINER`：重启容器
- `podman kill CONTAINER`：强制终止容器
- `podman pause CONTAINER`：暂停容器中的所有进程
- `podman unpause CONTAINER`：恢复容器中的所有进程
- `podman rm CONTAINER`：删除一个或多个容器
- `podman inspect CONTAINER`：显示容器的详细信息

## 镜像管理

- `podman images`：列出本地镜像
- `podman pull IMAGE`：从仓库拉取镜像
- `podman push IMAGE`：将镜像推送到仓库
- `podman build [OPTIONS] PATH`：使用 Dockerfile 构建镜像
- `podman rmi IMAGE`：删除一个或多个镜像
- `podman tag IMAGE[:TAG] TARGET_IMAGE[:TAG]`：为镜像添加标签
- `podman save IMAGE`：将镜像保存为 tar 文件
- `podman load`：从 tar 文件加载镜像

## 容器执行与交互

- `podman exec [OPTIONS] CONTAINER COMMAND [ARG...]`：在运行的容器中执行命令
- `podman attach CONTAINER`：连接到运行中的容器
- `podman logs [OPTIONS] CONTAINER`：获取容器的日志输出
- `podman ps [OPTIONS]`：列出容器

## Pod 管理（Podman 特有）

- `podman pod create [OPTIONS]`：创建一个新的 Pod
- `podman pod ls`：列出所有 Pods
- `podman pod rm POD`：删除一个或多个 Pods
- `podman pod inspect POD`：显示 Pod 的详细信息
- `podman pod logs POD`：获取 Pod 的日志

## 网络管理

- `podman network ls`：列出网络
- `podman network create [OPTIONS] NETWORK`：创建网络
- `podman network rm NETWORK`：删除网络
- `podman network inspect NETWORK`：显示网络详细信息

## 卷管理

- `podman volume ls`：列出卷
- `podman volume create [OPTIONS] VOLUME`：创建卷
- `podman volume rm VOLUME`：删除卷
- `podman volume inspect VOLUME`：显示卷的详细信息

## 系统相关

- `podman info`：显示系统信息
- `podman version`：显示版本信息
- `podman system prune`：清理未使用的数据
- `podman events`：获取实时事件

Podman 命令与 Docker 命令高度兼容，大多数情况下只需将 `docker` 替换为 `podman` 即可使用。
```
